# Система за управление на библиотека

Уеб приложение за управление на библиотека, разработено с ASP.NET Core 8.0 MVC и C#. Системата позволява разглеждане на каталог с книги, заемане и връщане на книги, както и пълно управление на съдържанието от страна на администратор.

## Функционалности

### Публична част
- Разглеждане на каталог с всички книги
- Преглед на детайли за всяка книга
- Търсене по заглавие или автор
- Филтриране по жанр

### Потребителска част
- Регистрация и вход в системата
- Заемане на книга с автоматичен 14-дневен срок за връщане
- Връщане на заета книга
- Преглед на история на заеманията със статус — активна, просрочена, върната

### Административен панел
- Пълно управление на книги (добавяне, редактиране, изтриване)
- Управление на автори
- Управление на жанрове
- Защита срещу изтриване на свързани записи

## Технологии

| Технология | Версия | Предназначение |

## Стартиране на проекта

### 1. Клониране на репото

Отворете Command Prompt или Git Bash и изпълнете:

```
git clone https://github.com/AdrianVittig/LibrarySystem.git
```

### 2. Отваряне във Visual Studio

Навигирайте до клонираната папка и отворете файла `LibrarySystem.slnx` с Visual Studio 2022.

### 3. Прикачване на базата данни

Базата данни се намира в папката `LibrarySystem/App_Data/`. За да я прикачите:

1. Отворете **View → SQL Server Object Explorer** във Visual Studio
2. Десен бутон върху **(localdb)\MSSQLLocalDB → Databases**
3. Изберете **Add Database...**
4. Навигирайте до `LibrarySystem/App_Data/LibrarySystemDb.mdf`
5. Натиснете **Open**

### 4. Стартиране

Поставете `LibrarySystem` като Startup Project (десен бутон върху проекта → **Set as Startup Project**), след което натиснете **F5**.

Приложението ще се отвори автоматично в браузъра на адрес `https://localhost:7008`.

## Unit тестове

Проектът съдържа 8 unit теста, които покриват критичната бизнес логика. За да ги стартирате:

- В Visual Studio изберете **Test → Run All Tests**
- Резултатите ще се покажат в Test Explorer прозореца

Тестовете използват:
- **xUnit** — framework за тестване
- **Moq** — за създаване на mock обекти
- **EntityFrameworkCore.InMemory** — изолирана база данни за всеки тест

## Структура на проекта

- **Controllers/** — HTTP заявки и бизнес логика
- **Data/** — DbContext и DbSeeder
- **Migrations/** — EF Core миграции
- **Models/** — Класове за базата данни
- **ViewModels/** — Класове за форми
- **Views/** — Razor view файлове
- **wwwroot/** — CSS, JS, статични файлове
- **App_Data/** — База данни (.mdf, .ldf)
- **Program.cs** — Конфигурация на приложението
